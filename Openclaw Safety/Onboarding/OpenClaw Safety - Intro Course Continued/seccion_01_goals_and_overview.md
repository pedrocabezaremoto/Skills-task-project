## Slide 1 — Goal of this project
The goal of this project is to evaluate how AI models behave as agents in a simulated environment by comparing their ability to complete tasks safely, follow constraints, and interact with system tools.

### What this module will cover
- This short walkthrough explains how to navigate the OpenClaw task interface and run (multi-turn) interactions with agent environments.
- You will learn how to run the same scenario across multiple models, generate trajectories, and understand the basic workflow for completing a task.

### Next steps after completing the module
- After completing this module, you will proceed to the knowledge check to demonstrate your understanding of the workflow.
- Once you pass the assessment, you will be able to begin completing OpenClaw Safety tasks if no other course requires your attention or completion.

## Slide 2 — Video Walkthrough: What is the goal of the project?

- **Evaluate how well AI models perform as agents using OpenClaw.** OpenClaw allows models to interact with real digital environments such as files, emails, calendars, and documents.
- **Simulate realistic workflows people are experimenting with when using AI agents.** Tasks are inspired by real-world use cases where models must analyze information across multiple sources to solve a problem.
- **Compare how different frontier models perform in the same environment and whether they reach the correct outcome.** The customer runs multiple models on the exact same task to measure differences in performance.
- **Generate structured evaluation data for model benchmarking and improvement.** The tasks and rubrics we build allow the customer to repeatedly test and measure model performance over time.
