I work with audio data and I want to build a simple audio fingerprinting and matching library in Python that can identify short audio clips by comparing them against a database of known recordings, similar in concept to Shazam but much simpler. The fingerprinting process takes a WAV audio file, computes its spectrogram using a short time Fourier transform (STFT) with overlapping windows, finds spectral peaks in each time frame by detecting local maxima that exceed a threshold above the mean energy, and creates fingerprint hashes from pairs of peaks within a configurable time distance, where each hash encodes the frequencies of both peaks and the time difference between them. These hashes along with their time offsets are stored in a SQLite database keyed by the hash value. To identify an unknown clip, the same fingerprinting process runs on the clip, the resulting hashes are looked up in the database, and for each matching hash the time offset difference between the clip's hash and the database entry's hash is computed. If many hashes from the same recording all show the same time offset difference (indicating temporal alignment), that recording is identified as the match. The match confidence is the count of aligned hashes. The system should work even with noisy clips or clips starting from different points in the song. Include a reference database builder that fingerprints a directory of WAV files, a query command that takes a short WAV clip and returns the best matching recording with confidence score, and a test mode that takes a known recording, extracts a random 10 second segment with added noise, and verifies the system can still identify it correctly.
