# Phase 3: Expert Rubrics

## Objective
Create rubric criteria to cover architectural constraints and qualitative requirements that unit tests cannot verify.

## Inputs
- `inputs/prompt-rewritten.md` — Technical specification
- `inputs/test-coverage-gaps.md` — Requirements not covered by tests

## Outputs
- `outputs/rubrics.md` — Final rubric file (5–30 criteria)
- `outputs/coverage-audit.json` — 100% requirement coverage map

## Rubric Standards
- Atomic: one discrete requirement per rubric
- Self-contained: no external context needed
- Implementation agnostic: accepts any valid solution
- Positive framing: affirmative pass condition
- Weights: ONLY 1, 3, or 5
