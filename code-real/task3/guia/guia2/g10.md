the

Overview  
The  scenario  generation  process  is  designed  to  set  up  a  situation  that  an  agent  will  
subsequently
 
resolve.
 
You
 
need
 
to
 
provide
 
two
 
key
 
inputs:
 
1.  Scenario  Description:  This  establishes  the  initial  conditions  and  setup  for  the  overall  
task.
 2.  Task  Prompt:  This  is  the  actual  instruction,  written  from  the  persona's  point  of  view,  that  
guides
 
the
 
agent
 
to
 
investigate
 
what
 
occurred,
 
discover
 
the
 
situation,
 
and
 
take
 
the
 
necessary
 
actions
 
for
 
resolution.
 
Workflow  
The  process  for  scenario  generation  involves  the  following  steps:  
1.  Draft  Scenario  Description:  Write  a  description  outlining  only  the  initial  conditions  of  
the
 
scenario
 
based
 
on
 
the
 
task
 
category
 
and
 
your
 
assigned
 
Business
 
Function
 
and
 
Persona.
 2.  Generate  Artifacts:  Execute  the  pipeline  to  create  new  artifacts  for  the  described  
scenario.
 3.  Evaluate  Artifacts:  Review  the  artifacts  that  were  generated.  4.  Develop  Task  Prompt:  Formulate  the  final  task  prompt.  
Step  1:  Understand  Your  Inputs
 
1.  Review  the  Persona:  Start  with  the  provided  persona  (e.g.,  "Chloe  Vance  -  Operations  
Manager").
 
Carefully
 
read
 
the
 
brief
 
detailing
 
their
 
role,
 
recent
 
actions,
 
and
 
identity.
 2.  Develop  a  Plausible  Scenario:  The  scenario  you  generate  must  be  realistic  for  the  
persona's
 
role
 
and
 
adhere
 
to
 
everything
 
else.
 3.  Incorporate  Universe  References:  Use  the  universe  reference  sheet  to  include  
authentic
 
details
 
like
 
names
 
of
 
people,
 
companies,
 
and
 
vendors.
 
Step  2:  Write  the  Scenario  Description
 
Your  scenario  description  should  detail  the  initial  conditions  -  the  artifacts  present  when  the  
persona
 
first
 
encounters
 
the
 
situation.
