What  Are  Rubrics?  
Rubrics  are  specific,  checkable  statements  about  what  the  AI  agent  should  (and  should  not)  
accomplish.
 
Each
 
rubric
 
is
 
a
 
simple
 
yes-or-no
 
claim.
 
An
 
LLM
 
judge
 
reads
 
the
 
agent's
 
trajectory
 
and
 
checks
 
each
 
rubric
 
against
 
what
 
it
 
finds
 
there.
 
You  write  the  rubrics.  The  judge  reads  the  trajectory.  Pass  or  fail.

●  The  original  task  prompt  —  what  the  agent  was  asked  to  do  ●  The  agent's  trajectory  —  every  tool  call,  its  parameters,  its  response,  and  the  agent's  
reasoning
 
at
 
each
 
step
 ●  The  agent's  final  response  —  the  summary  text  at  the  end  ●  Your  rubric  criteria

Three  Rubric  Categories  
Every  rubric  belongs  to  one  of  three  categories:  Tool  Selection  (TS),  Query  Construction  (QC),  
and
 
Outcome.
 
These
 
categories
 
provide
 
downstream
 
ML
 
teams
 
with
 
labeled
 
training
 
signals.
 
Core  rule:  Write  Outcome  rubrics  first.  Then  add  TS  and  QC  only  for  what  Outcome  
doesn't
 
already
 
cover.
 
 
Category  1:  Tool  Selection  (TS)  
Read/lookup  tool  calls  only  —  never  write  actions.  
What  it  checks:  Did  the  agent  use  the  right  tool?  
Write  a  TS  rubric  only  when  the  agent  might  plausibly  skip  the  tool.  Not  all  tools  have  equal  
discriminating
 
power
 
—
 
see
 
the
 
Default
 
vs
 
Non-Default
 
Tool
 
List
 
below.
 
●  Skip  TS  for  default  tools  —  the  agent  will  always  use  these,  so  a  TS  rubric  provides  no  
signal
 ●  Write  TS  for  non-default  tools  —  the  agent  might  skip  these  entirely,  so  TS  has  real  
discriminating
 
power
 
For  each  non-default  tool,  write  one  TS  rubric  regardless  of  how  many  times  the  agent  calls  it.  
Calling
 crm_search_contacts three  times  still  only  warrants  one  TS  rubric.

crm_search_companies).
 
When  multiple  non-default  tools  could  achieve  the  same  goal,  use  "or"  to  list  valid  alternatives.
