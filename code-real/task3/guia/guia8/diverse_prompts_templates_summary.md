## Purpose

MoveOps tasks should reflect **real business workflows** — not just "research and email."  
This guide provides 14 proven task patterns across 5 business functions, each with example prompts and expected write actions.

---

## Core Instructions

### Use it to diversify
- If all your tasks look the same, come back here
- Each category drives **different write tools**, **different services**, and **different kinds of agent reasoning**
- Tasks should span the full matrix over time — not cluster in one pattern

### Follow the patterns, not just the examples
- Example prompts are **illustrations**, not templates to copy
- Use them to understand what a rich, multi-action task looks like in each category
- Then build your own from what you find in your universe

---

## What to Expect in the Full Guide (from Gist)
- 14 task patterns organized across 5 business functions
- Each pattern includes:
  - Example prompt
  - Expected write actions (which tools the agent should call)
- Designed to prevent task clustering and ensure tool/service diversity

---

## Key Takeaway
> Diversity = different write tools + different services + different reasoning patterns.  
> A task portfolio that only triggers `send_email` is not diverse enough.
